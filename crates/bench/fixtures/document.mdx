# Getting Started with MDX

MDX lets you use **JSX** in your *Markdown* content. You can import
components and render them inline, like <Callout type="info">a small note</Callout>,
or as block-level elements.

## Why MDX?

- Write content in familiar Markdown
- Drop in interactive components where you need them
- Keep everything in one file with `import` and `export`

<Callout type="warning">
  Components used in flow position can contain **their own** Markdown,
  including [links](https://example.com) and `inline code`.
</Callout>

### A quick example

Here is a chart rendered from inline data:

<Chart
  values={[1, 2, 3, 5, 8, 13, 21]}
  labels={['a', 'b', 'c', 'd', 'e', 'f', 'g']}
  options={{ animate: true, color: meta.author }}
/>

The expression {2 + 2} evaluates at build time, and so does {meta.title ?? 'Untitled'}.
